01

What is Power BI?

Power BI is Microsoft's business intelligence platform — a suite of tools that lets you connect to data, transform it, build data models, and create interactive visual reports and dashboards.

It sits at the intersection of Excel's familiarity and enterprise-grade analytics. Widely used by analysts, data teams, and business users across industries.

Self-service BI Data Modeling Interactive Dashboards Cloud + Desktop Microsoft Ecosystem
02

The Three Core Components

Power BI Desktop

Free Windows app where you build everything — connect data, transform it, model it, and design reports.

Power BI Service

Cloud platform (app.powerbi.com) for publishing, sharing, and collaborating on reports and dashboards.

Power BI Mobile

iOS/Android app to view and interact with published reports on the go.

Power BI Gateway

Bridge that keeps cloud datasets refreshed from on-premise data sources automatically.

03

The Core Workflow

1 · Connect to Data
Use Get Data to connect to 100+ sources — Excel, SQL Server, Sharepoint, REST APIs, Salesforce, Azure, Google Analytics, and more.
2 · Transform with Power Query
The Power Query Editor is a visual ETL tool. Clean nulls, rename columns, merge tables, unpivot, change types — all tracked as repeatable steps.
3 · Build a Data Model
Define relationships between tables in the Model view. One-to-many, many-to-many. This is where star schemas live.
4 · Write DAX Measures
Create calculated measures and columns using DAX (Data Analysis Expressions) — aggregations, ratios, running totals, and time intelligence.
5 · Design Reports
Drag fields onto a canvas to create bar charts, tables, maps, cards, slicers. All visuals cross-filter each other by default.
6 · Publish & Share
Publish to Power BI Service. Share via links, embed in Teams/SharePoint, schedule dataset refreshes, or create Dashboards by pinning visuals.
04

Power Query (M Language)

Power Query records every transformation as a step, generating M code behind the scenes. Most transformations are done through the GUI — you rarely need to write M manually.

Key operations: Remove duplicates, filter rows, split columns, merge/append queries, pivot/unpivot, group by, add custom columns, change data types.

// Example: Custom column formula in M
let
  Source = Excel.Workbook(File.Contents("data.xlsx")),
  Sheet = Source{[Name="Sales"]}[Data],
  Typed = Table.TransformColumnTypes(Sheet,
    {{"Date", type date}, {"Amount", type number}}),
  Filtered = Table.SelectRows(Typed, each [Amount] > 0)
in
  Filtered
→ Keep Power Query for shaping data. Don't do calculations here — save that for DAX measures in the model.

Data Modeling Basics

A good model is everything. The Star Schema is the gold standard — one central fact table surrounded by dimension tables.

Fact Tables

Store transactions/events. High row count. Numeric measures (sales, quantity, cost). Example: FactSales.

Dimension Tables

Descriptive context. Low row count. Text attributes. Example: DimDate, DimCustomer, DimProduct.

→ Always create a separate Date table marked as a date table. Use CALENDARAUTO() or a custom M query. This unlocks all time intelligence functions.

Visuals & Report Design

The report canvas uses a drag-and-drop interface. Visuals auto-cross-filter on click. Slicers filter everything on the page.

Bar / Column Line Chart Card / KPI Matrix / Table Map (Filled/Bubble) Treemap / Donut Decomposition Tree Key Influencers Custom AppSource Visuals

Report vs Dashboard: Reports are multi-page, interactive, built in Desktop. Dashboards (in Service) are single-page, pin-based, and update in real time.

Drill-through lets users right-click a data point and navigate to a detail page filtered to that value. Bookmarks capture page state for interactive storytelling.

Power BI Service & Sharing

Workspaces
Team containers for reports, datasets, and dashboards. My Workspace is personal; create shared workspaces for collaboration.
Scheduled Refresh
Set datasets to auto-refresh up to 8x/day (Pro) or 48x/day (Premium). Requires a Gateway for on-prem sources.
Row-Level Security (RLS)
Restrict which data users see by defining DAX-based roles. A sales rep sees only their own region's data.
Apps
Package a workspace's content into a polished App for distribution to a large audience without giving them edit access.

10

Pro Tips to Level Up Fast

Use Import mode by default
Data is cached in-memory for fast performance. Use DirectQuery only when data must be real-time and the source can handle the load.
Never use VLOOKUP logic — use relationships
Properly related tables + RELATED() beats adding lookup columns. It's faster, cleaner, and more flexible.
Learn filter context vs row context
The single most important DAX concept. Understanding when a formula evaluates row-by-row vs. at an aggregate level unlocks everything.
Use measures, not calculated columns, for aggregations
Measures are computed at query time in filter context. Calculated columns are computed at refresh and bloat your model size.
Enable Performance Analyzer
View > Performance Analyzer shows how long each visual takes to render. Spot and fix slow DAX queries immediately.
